# Code of Conduct

## Purpose

This project is maintained as a practical, technical collaboration space.

The goal is straightforward:

- improve the downloader
- keep the codebase usable and reviewable
- discuss changes directly without personal attacks or noise

## Expected Behavior

Contributors, maintainers, and users should:

- stay factual and technical
- criticize code, behavior, and decisions rather than people
- explain disagreements clearly and concretely
- respect different experience levels
- keep discussions relevant to the repository
- report bugs, regressions, and risks honestly

Direct feedback is welcome. Hostility is not.

## Unacceptable Behavior

The following is not acceptable in project spaces, issues, discussions, pull requests, or other collaboration channels:

- harassment, threats, or intimidation
- personal insults, slurs, or discriminatory language
- repeated bad-faith arguments or deliberate disruption
- doxxing or sharing private information without permission
- sexualized language or imagery unrelated to the project
- spam, trolling, or intentionally derailing technical discussions
- encouraging unsafe, destructive, or malicious use of the project

## Collaboration Standards

This repository is opinionated about engineering quality.

That means:

- changes may be challenged if they are risky, unclear, or poorly justified
- maintainers may ask for narrower scope, better testing, or cleaner documentation
- disagreement is fine, but it must remain constructive

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or contributions that violate this Code of Conduct.

Depending on severity, maintainers may:

- request a behavioral correction
- lock or close discussions
- reject contributions
- temporarily or permanently block participation

## Reporting

If someone is violating this Code of Conduct, report it privately to the project maintainer or repository owner instead of escalating publicly.

When reporting, include:

- what happened
- where it happened
- links or screenshots if relevant
- any context needed to evaluate the situation

Reports will be reviewed in good faith and handled as discretely as possible.

## Project-Specific Note

This project includes automation, download tooling, and server-oriented workflows. Keep discussion and contributions focused on legitimate software improvement, deployment, maintenance, and user experience.

## Scope

This Code of Conduct applies to:

- the repository
- issues
- pull requests
- discussions
- review comments
- project-linked communication spaces related to this repository
